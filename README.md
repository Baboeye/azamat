# Material Accounting System 📋

Система управления материалами с функциональностью учета товаров, контрагентов и производства спецодежды для нефтегазовой промышленности.

## 🚀 Развертывание на Railway

### Предварительные требования
- GitHub аккаунт
- Railway аккаунт (https://railway.app)

### Шаги деплоя

1. **Подключите репозиторий**
   - Зайдите в Railway Dashboard
   - Создайте новый проект
   - Выберите "Deploy from GitHub"
   - Подключите репозиторий `azamat`

2. **Добавьте PostgreSQL базу данных**
   - В Railway Dashboard нажмите "Add Service"
   - Выберите PostgreSQL
   - Railway автоматически создаст переменную `DATABASE_URL`

3. **Настройте переменные окружения**
   - Перейдите в Variables вашего web-сервиса
   - Добавьте следующие переменные:
     ```
     SECRET_KEY=<очень-длинный-случайный-ключ>
     DEBUG=False
     ALLOWED_HOSTS=<ваш-домен>.up.railway.app
     CSRF_TRUSTED_ORIGINS=https://<ваш-домен>.up.railway.app
     ```

4. **Деплойте приложение**
   - Railway автоматически запустит `release.sh` перед стартом приложения
   - Будут выполнены миграции, собрана статика, инициализированы тестовые данные

### 📝 Тестовый аккаунт

После деплоя система инициализируется с тестовым суперпользователем:

```
Логин: azama
Пароль: azamat
```

Этот аккаунт имеет полные права администратора.

### 🎯 Функциональность

- **Управление материалами** - каталог, остатки, цены
- **Контрагенты** - поставщики и заказчики
- **Приемки и отпуски** - учет движения материалов
- **Производство** - шаблоны изделий, нормы расходов
- **Готовая продукция** - управление изделиями, отправки
- **Отчеты** - аналитика по остаткам и продаж
- **Аудит** - логирование всех операций

### 📦 Структура проекта

```
material_accounting/
```

### 🔧 Локальное развитие

```bash
# Клонирование репозитория
git clone https://github.com/Baboeye/azamat.git
cd azamat

# Создание виртуального окружения
python -m venv venv
source venv/bin/activate  # или venv\Scripts\activate на Windows

# Установка зависимостей
pip install -r requirements.txt

# Миграции
python manage.py migrate

# Инициализация тестовых данных
python manage.py init_data

# Запуск сервера
python manage.py runserver
```

Приложение будет доступно по адресу: http://localhost:8000

### 🗄️ Таблицы базы данных

- **Section** - секции склада
- **Contractor** - контрагенты (поставщики и заказчики)
- **Material** - каталог материалов
- **Receipt** - приемки материалов
- **Issue** - отпуски материалов
- **Scrap** - брак и отходы
- **AuditLog** - логирование операций
- **ProductCategory** - категории спецодежды
- **ProductTemplate** - шаблоны изделий
- **Product** - готовые изделия
- **ProductShipment** - отправки заказчикам
- **MaterialConsumption** - калькуляция расходов

- 4 категории спецодежды
- 2 шаблона производства
- 5 готовых изделий
- 3 приемки материалов
- 2 отправки готовой продукции

### 🛠️ Команды управления

```bash
# Создание миграций
python manage.py makemigrations

# Применение миграций
python manage.py migrate

# Сбор статических файлов
python manage.py collectstatic

# Инициализация данных
python manage.py init_data

# Создание суперпользователя
python manage.py createsuperuser

# Запуск тестов
python manage.py test
```
